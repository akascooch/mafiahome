# نقشه راه ویژگی‌ها

این سند شامل Issueهای پیشنهادی برای توسعه‌ی بخش‌های کلیدی پروژه است. برای هر ویژگی یک فایل Issue در پوشه [`docs/issues/`](issues/) ایجاد شده که حاوی شرح کامل، معیارهای پذیرش، Definition of Done، برآورد و نام برنچ پیشنهادی است. پس از ساخت Issue واقعی در GitHub، لینک‌ها را در همان فایل‌ها به‌روزرسانی کنید.
